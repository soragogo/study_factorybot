# Next.js 入門 01 - はじめてのNext.js 🐱

## 1. Next.jsとは？
**「React（エンジン）を使って、フルスタックWebアプリ（車）を作るためのフレームワーク」**

| 特徴 | React (単体) | Next.js |
| :--- | :--- | :--- |
| **役割** | UIライブラリ (部品を作る) | フレームワーク (アプリ全体を作る) |
| **レンダリング** | CSR (クライアント側) が基本 | **SSR (サーバー側)** が使えて高速・SEOに強い |
| **ルーティング** | 別途ライブラリが必要 | **ファイルシステムベース** (フォルダを作るだけ) |

## 2. 環境構築と起動
プロジェクトの作成には `create-next-app` を使う。

```bash
# 作成コマンド（全部入りパック）
npx create-next-app@latest

# 開発サーバー起動コマンド
npm run dev
# -> http://localhost:3000 でアクセス

```

> **Memo:** `npm run dev` は、プロジェクトフォルダ内の `node_modules` にある `next` コマンドを呼び出している。「ファイル変更の監視」と「自動更新」を行ってくれる。

## 3. 開発体験 (Fast Refresh)

* 開発中にファイルを保存すると、ブラウザをリロードしなくても**変更した部分だけが一瞬で書き換わる**。
* 状態（入力中の文字など）を保ったまま更新されるので、開発効率が良い。

## 4. ルーティング (Routing)

Next.js (App Router) では、設定ファイル不要でURLを作れる。

* `src/app/page.tsx` -> トップページ (`/`)
* `src/app/about/page.tsx` -> 自己紹介ページ (`/about`)

**フォルダ名がそのままURLになる！**

## 5. 画面遷移 (<Link> コンポーネント)

ページ移動には `<a>` タグではなく、Next.js専用の `<Link>` コンポーネントを使う。

```tsx
import Link from 'next/link';

<Link href="/about">自己紹介へ</Link>

```

* **`<a>` タグ**: ページ全体を再読み込みする（遅い・画面が白くなる）。
* **`<Link>`**: 必要な部分だけ書き換える（**爆速**・SPAの挙動）。
